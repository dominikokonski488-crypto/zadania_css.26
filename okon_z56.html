1) Składnia teoretyczna definiowania klasy

selektor.klasa { cecha: wartość }

2) Opis: klasa, "cecha" oraz "wartość"

Klasa to wartość atrybutu class="..." nadanego selektorowi z poziomu HTML.
"cecha" oraz "wartość" określają atrybuty elementu nadane przez style CSS.

3) Jaki znacznik może być selektorem? Trzy przykłady:

Selektorem może być dowolny znacznik HTML, np.:
p – akapit
h1 – tytuł pierwszego stopnia
td – komórka tabeli

4) Uwagi na temat nazwy klas

Nazwą może być dowolny pojedynczy wyraz, który nie może zawierać:
a) spacji
b) kropki, przecinka, dwukropka
...aż do...
h) znaku równości, plusa, nawiasów
Może zawierać:
a) litery A-Z, a-z
b) cyfry 0-9
c) myślniki (-)
d) podkreślniki (_)
Nazwa nie może zaczynać się od cyfry ani myślnika.
Gdy chcemy użyć znaków "zakazanych", należy poprzedzić je odwróconym ukośnikiem \.

5) Zastosowanie klas

Klasy są przydatne, gdy wiele elementów na różnych stronach ma takie samo formatowanie, a nie możemy użyć selektora typu, bo inne znaczniki tego samego rodzaju mają być inaczej sformatowane.

6) Kiedy ma zastosowanie lista klas?

Lista klas ma zastosowanie, gdy chcemy przypisać do jednego elementu kilka różnych klas jednocześnie:
<selektor class="klasa1 klasa2 klasa3">...</selektor>

7) Składnia listy klas i uwagi

<selektor class="klasa1 klasa2 klasa3">...</selektor>
Uwagi: klasy muszą być zdefiniowane w arkuszu stylów. Kolejność wpisywania nazw klas nie ma znaczenia – jeśli kilka klas odnosi się do tej samej własności, decyduje zasada kaskadowości.

8) Podzbiory klas – składnia

Podzbiór klas pozwala określić, jakie klasy musi mieć element jednocześnie:
selektor.klasa1.klasa3 { cecha: wartość }

Zmienia formatowanie elementu: <selektor class="klasa1 klasa2 klasa3">

9) Kiedy stosować klasy?

Gdy jakiś element o specyficznym formatowaniu występuje przynajmniej raz na niektórych stronach serwisu – jest to sygnał, aby umieścić deklarację jego klasy w zewnętrznym arkuszu stylów.

10) Kiedy planować klasy?

Już na etapie projektowania serwisu należy dokładnie przemyśleć, jakie klasy mogą się przydać. Im więcej klas w zewnętrznym arkuszu, tym łatwiejsza późniejsza modyfikacja stron.

11) Cel stosowania komentarzy w klasach

Komentarze przed każdą definicją klasy (znaki: /* ... */) opisują jej przeznaczenie. Po dłuższym czasie, gdy arkusz zawiera wiele klas, komentarze pomagają zapamiętać, do czego dana klasa służy.

12) Przykład definiowania i wywoływania klasy

Definicja w CSS:
p.przyklad_klasa { color: red; }
Wywołanie w HTML:
<p class="przyklad_klasa">To jest akapit.</p>

13) Klasa nieuniwersalna

Klasa nieuniwersalna odnosi się tylko do jednego selektora.
Definicja:
p.moja { color: green; font-size: 14pt; }
Wywołanie:
<p class="moja">Treść akapitu</p>
Taka klasa może być użyta tylko w znaczniku <p>, do którego została przypisana.

14) Klasa uniwersalna i znak *

Klasa uniwersalna pozwala przypisać własności do wszystkich elementów, niezależnie od typu znacznika.
Definicja z gwiazdką:
*.przyklad_uniwersalna { color: red; }
Definicja bez gwiazdki (gwiazdka jest domyślna):
.przyklad_uniwersalna { color: red; }
Wywołanie:
<p class="przyklad_uniwersalna">akapit</p>
<h1 class="przyklad_uniwersalna">nagłówek</h1>
Znak * oznacza "każdy element" i można go pominąć – efekt jest taki sam.
